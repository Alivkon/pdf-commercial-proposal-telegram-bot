# Оптимизация поиска компании клиента

## Проблема

При генерации коммерческого предложения для физического лица система выполняла ненужный поиск в Qdrant векторной базе данных, что приводило к:
- Лишним затратам времени (~500-1000мс на запрос)
- Дополнительным расходам на API OpenAI (генерация эмбеддинга)
- Избыточной нагрузке на Qdrant

## Решение

Добавлена проверка ключевых слов физического лица **перед** поиском в Qdrant.

### Код в src/index.js (строки 236-241)

```javascript
// Проверяем, является ли получатель физическим лицом
const isIndividual = /(физику|для\s+физика)/i.test(userText);

if (isIndividual) {
  console.log('\n[Тип получателя] Физическое лицо - поиск компании не требуется');
} else {
  // Выполняем поиск в Qdrant только для организаций
  const searchResults = await searchCompaniesInQdrant(userText, 3);
  // ...
}
```

## Ключевые слова

Регулярное выражение: `/(физику|для\s+физика)/i`

**Примеры срабатывания:**
- "КП физику" → физлицо
- "для физика на станцию" → физлицо
- "Подготовьте КП для физика" → физлицо

**Примеры НЕ срабатывания:**
- "КП для компании Энерком" → организация (поиск выполняется)
- "КП на зарядную станцию" → организация (поиск выполняется)

## Технические детали

### Почему не используется \b (word boundary)?

Граница слова `\b` в JavaScript не работает с кириллицей:

```javascript
// ❌ НЕ работает
/\b(физику|для\s+физика)\b/i.test('КП физику')  // false

// ✅ Работает
/(физику|для\s+физика)/i.test('КП физику')      // true
```

Это связано с тем, что `\b` определяется как граница между `\w` и `\W`, где `\w` = `[A-Za-z0-9_]` (без кириллицы).
