# Отчет о прохождении информации в системе обработки документов

## Общая схема потока данных

```
Telegram Bot → Buffer → Document Processor → ChatGPT → PostgreSQL
     ↓            ↓            ↓                  ↓          ↓
 Получение    Извлечение   Анализ AI        Структур.   Сохран.
  файла        текста     и валидация        данные      в БД
```

## Детальное прохождение информации

**Файл:** [src/index.js:340-405](src/index.js#L340-L405)

**Что происходит:**

### 4. Сохранение в PostgreSQL

**Файл:** [src/dbService.js](src/dbService.js)

**Таблицы:**
- `company` - информация о компаниях
- `company_document` - обработанные документы

**Что происходит:**
1. Вычисление SHA256 хеша файла для дедупликации
2. Поиск существующей компании по ИНН+КПП
3. Создание новой компании или обновление существующей
4. Сохранение документа с привязкой к компании

**Структура таблицы `company`:**
```sql
CREATE TABLE company (
  id uuid PRIMARY KEY,
  name_full text,
  name_short text,
  inn text NOT NULL,
  kpp text,
  ogrn text,
  okpo text,
  oktmo text,
  legal_address text,
  actual_address text,
  email text[],
  phone text[],
  ceo_name text,
  ceo_name_dative text,
  created_at timestamptz,
  updated_at timestamptz,
  UNIQUE (inn, kpp)
);
```

**Структура таблицы `company_document`:**
```sql
CREATE TABLE company_document (
  id uuid PRIMARY KEY,
  company_id uuid REFERENCES company(id),
  file_name text,
  file_sha256 text,
  storage_url text,
  raw_text text,
  extracted_json jsonb,
  confidence jsonb,
  model_version text,
  prompt_version text,
  created_at timestamptz
);
```

## Особенности обработки

### 1. Дедупликация
- Каждый файл хешируется (SHA256)
- Повторная загрузка того же файла → пропускается
- Опция `skipDuplicates: false` для принудительной обработки

### 2. Обновление данных
- При `updateExisting: true` - обновляет существующую компанию
- Сохраняет все предыдущие версии документов

### 3. Валидация
Автоматическая проверка:
- ИНН: 10 или 12 цифр
- КПП: 9 цифр
- ОГРН: 13 или 15 цифр
- Email: формат email
- Наличие обязательного ИНН

### 4. Метрики уверенности
ChatGPT возвращает уровень уверенности для каждого поля:
```json
{
  "confidence": {
    "inn": 0.95,
    "name": 0.90,
    "address": 0.85,
    "overall": 0.90
  }
}
```
