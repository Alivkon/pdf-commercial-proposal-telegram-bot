# 🌐 Веб-интерфейс базы компаний - Готово!

## ✅ Что было создано

В папке `web/` создан полноценный веб-интерфейс для просмотра компаний из PostgreSQL.

### 📁 Структура:

```
web/
├── index.html              # Веб-страница с таблицей компаний
├── server.js               # HTTP сервер + REST API
├── ecosystem.config.js     # Конфигурация для PM2
├── start.sh               # Скрипт быстрого запуска
├── logs/                  # Папка для логов
└── README.md              # Подробная документация
```

## 🚀 Быстрый старт

### Способ 1: Простой запуск (для теста)

```bash
cd web
node server.js
```

Откройте в браузере: http://localhost:3000

### Способ 2: С помощью скрипта

```bash
cd web
./start.sh
```

### Способ 3: С PM2 (рекомендуется для продакшна)

```bash
cd web
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

## 🎨 Возможности интерфейса

### ✨ Основные функции:

1. **Просмотр компаний**
   - Все компании из PostgreSQL в виде таблицы
   - Поля: Название, ИНН, КПП, ОГРН, Руководитель, Email, Дата

2. **Поиск в реальном времени**
   - Поиск по названию компании
   - Поиск по ИНН, КПП, ОГРН
   - Поиск по имени руководителя

3. **Сортировка**
   - Кликните на заголовок колонки для сортировки
   - Повторный клик меняет направление (↑/↓)

4. **Статистика**
   - Общее количество компаний
   - Количество документов в БД
   - Время последнего обновления

5. **Адаптивный дизайн**
   - Работает на мобильных устройствах
   - Красивый современный интерфейс

## 🌍 Деплой на сервер

### Вариант 1: PM2 (рекомендуется)

```bash
# 1. Запустите сервер
pm2 start web/ecosystem.config.js

# 2. Сохраните конфигурацию
pm2 save

# 3. Настройте автозапуск
pm2 startup

# 4. Проверьте статус
pm2 status
```

### Вариант 2: С Nginx (для продакшна)

1. Запустите сервер на порту 3000
2. Настройте Nginx:

```nginx
server {
    listen 80;
    server_name companies.your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

3. Перезагрузите Nginx:
```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 🔒 Безопасность

**⚠️ ВАЖНО:** По умолчанию интерфейс доступен всем!

### Добавить защиту паролем (через Nginx):

```bash
# Создайте файл паролей
sudo htpasswd -c /etc/nginx/.htpasswd admin

# Добавьте в конфигурацию Nginx
location / {
    auth_basic "Restricted Access";
    auth_basic_user_file /etc/nginx/.htpasswd;
    proxy_pass http://localhost:3000;
}
```

## 📝 Управление

### Команды PM2:

```bash
# Запуск
pm2 start web/ecosystem.config.js

# Остановка
pm2 stop elter-web

# Перезапуск
pm2 restart elter-web

# Логи
pm2 logs elter-web

# Статус
pm2 status

# Удаление
pm2 delete elter-web
```

### Прямой запуск:

```bash
# Запуск
cd web && ./start.sh

# Остановка
# Нажмите Ctrl+C

# Изменить порт
WEB_PORT=8080 node server.js
```

## 🐛 Решение проблем

### Порт 3000 занят

```bash
# Найдите процесс
lsof -i :3000

# Убейте процесс
kill -9 <PID>

# Или запустите на другом порту
WEB_PORT=8080 node server.js
```

### Ошибка подключения к БД

1. Проверьте `.env` файл
2. Убедитесь, что PostgreSQL запущен:
   ```bash
   systemctl status postgresql
   ```
3. Проверьте логи:
   ```bash
   pm2 logs elter-web
   ```
