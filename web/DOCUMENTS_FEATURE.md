# 📄 Функционал просмотра и загрузки документов

## Добавленные возможности

### 1. Просмотр всех документов
**Как использовать:**
- Наведите курсор на блок **"Документов"** в статистике (вверху страницы)
- Блок подсветится и появится курсор-указатель
- Кликните на блок — откроется всплывающее окно со списком всех документов из базы

**Что отображается:**
- Название файла
- Компания (название и ИНН)
- Размер файла
- Дата добавления
- Кнопка "Скачать"

### 2. Просмотр документов конкретной компании
**Как использовать:**
- Наведите курсор на любую строку таблицы компаний
- Строка подсветится фиолетовым цветом
- Кликните на строку — откроется окно с документами этой компании

**Что отображается:**
- Название компании в заголовке окна
- Список всех документов компании
- Размер и дата каждого документа
- Кнопка "Скачать" для каждого файла

### 3. Скачивание документов
**Как скачать:**
- В открытом окне документов нажмите кнопку **"⬇ Скачать"**
- Файл автоматически загрузится через браузер
- Сохранится с оригинальным именем

### 4. Управление всплывающим окном
**Закрыть окно можно:**
- Нажать кнопку **"×"** в правом верхнем углу
- Кликнуть вне окна (на затемненную область)
- Нажать клавишу ESC (опционально, можно добавить)

## Технические детали

### API endpoints

#### GET `/api/documents`
Получить все документы из базы данных

**Ответ:**
```json
[
  {
    "id": "uuid",
    "file_name": "document.pdf",
    "file_path": "/path/to/file.pdf",
    "file_type": "application/pdf",
    "file_size": 102400,
    "created_at": "2025-12-29T10:00:00Z",
    "name_short": "ООО «Компания»",
    "name_full": "Общество с ограниченной ответственностью «Компания»",
    "inn": "1234567890"
  }
]
```

#### GET `/api/documents/company/{companyId}`
Получить документы конкретной компании

**Параметры:**
- `companyId` - UUID компании

**Ответ:**
```json
[
  {
    "id": "uuid",
    "file_name": "document.pdf",
    "file_path": "/path/to/file.pdf",
    "file_type": "application/pdf",
    "file_size": 102400,
    "created_at": "2025-12-29T10:00:00Z"
  }
]
```

#### GET `/api/download/{documentId}`
Скачать документ

**Параметры:**
- `documentId` - UUID документа

**Ответ:**
- Бинарный файл с заголовками для скачивания
- `Content-Disposition: attachment; filename="..."`
- Поддерживаемые типы: PDF, DOC, DOCX

## Стили и UI

### Всплывающее окно
- **Размер:** 400-600px ширина, максимум 500px высота
- **Позиция:**
  - При клике на статистику — под блоком "Документов"
  - При клике на компанию — по центру экрана
- **Скролл:** Автоматически при большом количестве документов
- **Анимация:** Плавное появление/скрытие

### Мобильная адаптация
На экранах < 768px:
- Окно занимает 90% ширины экрана
- Кнопки "Скачать" растягиваются на всю ширину
- Вертикальное расположение элементов

### Интерактивность
- Строки таблиц подсвечиваются при наведении
- Блок статистики "Документов" реагирует на наведение
- Элементы списка документов анимируются
- Курсор-указатель на кликабельных элементах

## Безопасность

### Валидация
- Проверка существования файла перед отдачей
- Проверка UUID параметров
- Защита от path traversal атак через `path.resolve()`

### Обработка ошибок
- 404 если документ не найден
- 500 при ошибках чтения файла
- Логирование всех ошибок в консоль

## Использование

### Запуск сервера
```bash
cd web
node server.js
```

1. **Посмотреть все документы:**
   - Кликните на "Документов" (число в статистике)

2. **Найти документы конкретной компании:**
   - Найдите компанию через поиск
   - Кликните на строку компании

3. **Скачать документ:**
   - Откройте список документов (любым способом)
   - Нажмите "⬇ Скачать" у нужного файла

## Технические изменения

### Изоляция от зависимостей бота

Для веб-интерфейса создан отдельный модуль `web/database.js`, который:
- Подключается к PostgreSQL напрямую
- Не импортирует модули бота (OpenAI, Qdrant и т.д.)
- Использует только необходимые зависимости (pg, dotenv)

**Файлы:**
- `web/database.js` - изолированный модуль БД
- `web/server.js` - использует `./database` вместо `../src/database.service`

### Структура таблицы documents

В базе данных используется:
- `storage_url` - путь к файлу (может быть NULL для старых документов)
- `file_sha256` - хеш файла
- `file_name` - оригинальное имя файла
- Нет полей `file_size` и `file_type` (определяются динамически)

### Хранение файлов

**С 2025-12-29** все новые документы сохраняются на диск:
- Папка: `uploads/documents/`
- Именование: `{sha256_hash}{расширение}`
- Путь сохраняется в `storage_url`

**Старые документы** (до 2025-12-29):
- `storage_url` = NULL
- Кнопка "Скачать" заменяется на "📝 Только данные"
- Доступны только метаданные и извлеченный текст
