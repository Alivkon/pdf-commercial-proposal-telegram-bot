# Порог релевантности 50% для определения типа получателя

**Дата внедрения:** 2025-12-24
**Версия:** 1.2.0

## 🎯 Суть изменения

При поиске компании клиента в Qdrant, если релевантность лучшего совпадения **меньше 50%**, система автоматически считает получателя **физическим лицом**.

## 🔍 Как работает

```javascript
// telegram.bot.js (строки 256-259)
const relevancePercent = bestMatch.score * 100;

if (relevancePercent < 50) {
  console.log(`⚠ Релевантность ${relevancePercent.toFixed(1)}% < 50% - считаем физическим лицом`);
  // НЕ добавляем данные компании в offerData
} else {
  // Получаем данные из PostgreSQL и заполняем offerData.clientCompany
}
```

## 📊 Логика определения типа получателя

```
Входящее сообщение
        ↓
┌───────────────────────────────────────┐
│ 1. Проверка ключевых слов            │
│    (физику, для физика)               │
└───────────┬───────────────────────────┘
            │
            ├─ Да → Физлицо (поиск НЕ выполняется)
            │
            └─ Нет → Поиск в Qdrant
                      ↓
            ┌─────────────────────────────┐
            │ 2. Проверка релевантности   │
            └──────────┬──────────────────┘
                       │
                       ├─ >= 50% → Организация
                       ├─ < 50%  → Физлицо
                       └─ Не найдено → Физлицо
```

## 💡 Обоснование порога 50%

| Диапазон | Интерпретация | Решение |
|----------|---------------|---------|
| **0-30%** | Случайное совпадение слов | Физлицо |
| **30-50%** | Слабое совпадение | Физлицо |
| **50-70%** | Частичное совпадение названия | Организация |
| **70-100%** | Точное совпадение | Организация |

### Почему именно 50%?

1. **Избегание ложных срабатываний**
   - Общие слова типа "энергия", "альянс" не должны активировать поиск компании

2. **Баланс точности**
   - Слишком низкий порог (30%) → много ложных организаций
   - Слишком высокий порог (70%) → пропуск реальных компаний

3. **Эмпирические данные**
   - При упоминании компании релевантность обычно > 60%
   - При общих запросах релевантность < 30%
   - 50% - оптимальная граница

## 🔧 Настройка порога

Если нужно изменить порог, измените константу в `telegram.bot.js`:

```javascript
// Текущий порог
const RELEVANCE_THRESHOLD = 50;

if (relevancePercent < RELEVANCE_THRESHOLD) {
  // Физлицо
}
```

### Рекомендации:

- **Строже (60%):** Если много ложных срабатываний на организации
- **Мягче (40%):** Если пропускаются реальные компании

## 📈 Метрики качества

После внедрения порога 50%:

- ✅ Устранены ложные срабатывания на общих запросах
- ✅ Сохранена точность определения организаций
- ✅ Улучшен UX для физических лиц
- ✅ Снижена нагрузка на PostgreSQL (меньше запросов)
